# AI Workflow

AI tools were used throughout the WeHeartComics build as a practical development multiplier.

## How AI Was Used

### Product Thinking

AI helped explore product direction, feature scope, page structure, and user journeys for collectors.

Examples:

- Turning broad ideas into product flows
- Comparing SEO strategies
- Prioritizing which pages should be canonical
- Writing clearer collector-facing copy

### Engineering

AI helped implement and revise:

- Next.js routes and pages
- React components
- Prisma queries
- Admin routes
- Data import scripts
- Smoke tests and audits
- Redirects, metadata, and sitemap logic

### Debugging And QA

AI helped inspect errors, reason through build failures, check routes, review diffs, and verify behavior after changes.

### Content Modeling

AI helped define reusable content models for:

- Character guides
- First appearance sections
- Key issue explanations
- Beginner reading sections
- Collector notes
- SEO titles and descriptions

### SEO

AI helped identify that large groups of thin generated pages could hurt crawl prioritization. This led to a strategy centered on fewer, stronger canonical pages.

## What AI Did Not Replace

AI did not replace product judgment, taste, or ownership. I used it as a collaborator to move faster, challenge assumptions, generate options, and handle implementation details more efficiently.

The final direction, prioritization, and quality bar were human-led.

## Takeaway

The biggest advantage was not any single generated code snippet. The advantage was being able to work across product, engineering, data, QA, and growth with the leverage of a larger team.
