# Interview Notes

## Short Pitch

WeHeartComics is a comic collector intelligence platform that helps users research first appearances, key issues, character guides, reading paths, and market-aware collecting workflows.

I built it end to end with AI-assisted development workflows, using AI as a product, engineering, data, QA, and SEO collaborator.

## What To Highlight

- Built and deployed a real full-stack product.
- Designed data models for collector workflows and catalog research.
- Worked with messy third-party comic data.
- Built scripts for imports, migrations, cover caching, and audits.
- Improved SEO architecture by consolidating thin generated pages.
- Used AI tools thoughtfully to increase velocity without giving up product judgment.

## Technical Talking Points

- Next.js App Router implementation
- Prisma with PostgreSQL and Turso/libSQL
- Vercel deployment
- Vercel Blob asset handling
- Search and character guide architecture
- Canonical URLs, redirects, sitemap strategy, and noindex rules
- Admin tooling for maintaining catalog quality

## AI Talking Point

I used AI the way a small product team might divide work: one mode for implementation, one for code review, one for data cleanup, one for SEO strategy, and one for UX critique. That let me move faster while still owning the decisions and final product quality.
